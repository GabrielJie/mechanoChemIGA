2  * @page example3 Example 3 : Cahn-Hilliard (two species)
3  * \dontinclude CahnHilliard_twoSpecies/2D/userFunctions.cc
4  *
5  * This example implements the Cahn-Hilliard equation for phase-field modeling of a two species,
6  * as described by the following weak form of the PDE. The scalar fields are composition one, \f$c_1\f$,
7  * and composition two, \f$c_2\f$.
8  * Note the application of the higher-order Dirichlet boundary conditions \f$\nabla c_i\cdot\boldsymbol{n}=0\f$
9  * using Nitsche's method.
10  *
11  * Cahn-Hilliard:
12  *
13  * \f{eqnarray*}{
14  * 0 &=& \int_\Omega \left(w_1\frac{c_1 - c_{1_{prev}}}{\mathrm{d}t} +
15  * M\left(\nabla w_1\cdot(f_{,c_1c_1}\nabla c_1) + f_{,c_1c_2}\nabla c_2) + \kappa_1\nabla^2 w_1\nabla^2 c_1\right)\right) dV\\
16  * &\phantom{=}& - \int_{\partial\Omega} \left(w_1j_n +
17  * M\kappa_1\left(\nabla^2c_1(\nabla w_1\cdot\boldsymbol{n}) + \nabla^2w_1(\nabla c_1\cdot\boldsymbol{n})\right)
18  * - \tau(\nabla w_1\cdot\boldsymbol{n})(\nabla c_1\cdot\boldsymbol{n})\right) dS\\
19  * 0 &=& \int_\Omega \left(w_2\frac{c_2 - c_{2_{prev}}}{\mathrm{d}t} +
20  * M\left(\nabla w_2\cdot(f_{,c_2c_1}\nabla c_1) + f_{,c_2c_2}\nabla c_2) + \kappa_1\nabla^2 w_2\nabla^2 c_2\right)\right) dV\\
21  * &\phantom{=}& - \int_{\partial\Omega} \left(w_2j_n +
22  * M\kappa_1\left(\nabla^2c_2(\nabla w_2\cdot\boldsymbol{n}) + \nabla^2w_2(\nabla c_2\cdot\boldsymbol{n})\right)
23  * - \tau(\nabla w_2\cdot\boldsymbol{n})(\nabla c_2\cdot\boldsymbol{n})\right) dS
24  * \f}
25  *
26  * Free energy density:
27  *
28  * \f{eqnarray*}{
29  * f(c_1,c_2) = c_2(c_1-0.1)^2(c_1-0.9)^2 + (c_2-0.2)^2(c_2-0.8)^2
30  * \f}
31  *
32  * The current settings prescribe random initial conditions and zero-flux
33  * boundary conditions. With these settings, the following evolution
34  * of the concentration of the two species is obtained:
35  *
36  * \htmlonly <style>div.image img[src="example3.png"]{width:20cm;}</style> \endhtmlonly
37  * @image html example3.png
38  *
39  * Implementation: Level 1 users
40  * ==============================
41  *
42  * To implement this model, we will specify the following through defining user functions: <br>
43  * - Initial conditions <br>
44  * - Constitutive model (via free energy density functions) <br>
45  * - Parameter values <br>
46  * - Weak form of the PDEs <br>
47  *
48  * First, we include the header file declaring the required user functions. These functions will be defined in this file.
49  *
50  * \line userFunctions
51  *
52  * Now, we first define any optional user functions. Optional user functions have a default definition that can be
53  * redefined by the user using a function pointer.
54  * This will be done in the \c defineParameters function. The available list of optional user functions includes:
55  * \c boundaryConditions, \c scalarInitialConditions, \c vectorInitialConditions, \c loadStep, \c adaptiveTimeStep, and \c projectFields.
56  * In this example, we redefine only the \c scalarInitialConditions function, while using the default functions for the others.
57  *
58  * <b> The \c scalarInitialConditions function </b>
59  *
60  * We initialized the two composition fields to be random about 0.5.
61  *
62  * \skip template
63  * \until //end
64  *
65  * <b> Free energy density derivative functions </b>
66  *
67  * This phase-field implementation requires the first and second derivatives of the chemical free energy density function
68  * \f$f(c_1,c_2) = c_2(c_1-0.1)^2(c_1-0.9)^2 + (c_2-0.2)^2(c_2-0.8)^2\f$. We define the function computing
69  * \f$\partial^2 f/\partial c \partial c\f$ here.
70  * Note that this free energy derivative function is used only in this file. It is not a member of any class,
71  * nor will we use it to set any function pointers.
72  *
73  * \skip template
74  * \until //end
75  *
76  * <b> The \c defineParameters function </b>
77  *
78  * The user is required to define the \c defineParameters and \c residual functions. The \c defineParameters defines variables
79  * and functions in the \c AppCtx object. The \c AppCtx object is defined
80  * in the appCtx.h file. This function is used to define any values in \c user that will be needed in the problem.
81  * It is also used to set any function pointers for user functions that we have redefined.
82  *
83  * Many of these values can be overwritten by the parameters.prm file, which we will look at later.
84  *
85  * \skip template
86  * \until void
87  *
88  * Here, we define the mesh by setting the number of elements in each direction, e.g. a 100x100 element mesh.
89  *
90  * \skip user.N[0]
91  * \until user.N[1]
92  *
93  * We also define the dimensions of the domain, e.g. a unit square.
94  *
95  * \skip user.L[0]
96  * \until user.L[1]
97  *
98  * We can define additional material parameters that are not explicity listed in the \c user structure by
99  * defining elements of the \c matParam C++ map, which maps \c std::string to \c double. These values can also be overwritten
100  * in the parameters file.
101  *
102  * \skip "mobility"
103  * \until "kappa"
104  *
105  * We define the initial time step and total simulation time. We also have the options to use restart files, in which case
106  * we would set the iteration index and time at which to start. We leave these values at zero to begin a new simulation.
107  * We also have the option to output results at regular intervals (e.g. every 5 time steps).
108  *
109  * \skip user.dtVal
110  * \until user.skipOutput
111  *
112  * We specify the number of vector and scalar solution and projection fields by adding the name of each field to
113  * their respective vector. Here, we have one scalar solution field (the composition).
114  * We do not use any vector solution fields or projection fields in this example.
115  *
116  * \skip "c1"
117  * \until "c2"
118  *
119  * We can specify the polynomial order of the basis splines, as well as the global continuity.
120  * Note that the global continuity must be less than the polynomial order.
121  * Here, we use quadratic basis functions with C-1 global continuity.
122  *
123  * \skip polyOrder
124  * \until globalContinuity
125  *
126  * Finally, we redirect the desired user function pointers to the \c scalarInitialConditions function that we
127  * defined above. This completes the \c defineParameters function.
128  *
129  * \skip scalarInitialConditions
130  * \until //end
131  *
132  * <b> The \c residual function </b>
133  *
134  * The residual function defines the residual that is to be driven to zero.
135  * This is the central function of the code.
136  * It is set up to follow the analytical weak form of the PDE.
137  * It has a number of arguments that give problem information at the current quadrature point.
138  *
139  * \skip template
140  * \until &r
141  *
142  * \c dV is a boolean, "true" if \c residual is being called for the volume integral and
143  * "false" if \c residual is being called for the surface integral.\n
144  * \c dS is a boolean, "false" if \c residual is being called for the volume integral and
145  * "true" if \c residual is being called for the surface integral.\n
146  * \c x gives the coordinates of the quadrature point.\n
147  * \c normal gives the unit normal for a surface quadrature point.\n
148  * \c c gives the information (values, gradients, etc.) for the scalar solution fields at the current quadrature point
149  * (see documentation for solutionScalars class).\n
150  * \c u gives the information (values, gradients, etc.) for the vector solution fields at the current quadrature point
151  * (see documentation for solutionVectors class).\n
152  * \c w1 gives the information for the scalar test functions.\n
153  * \c w2 gives the information for the vector test functions.\n
154  * \c user is a structure available for parameters related to the initial boundary value problem (e.g. elasticity tensor).\n
155  * \c r stores the scalar value of the residual for the weak form of the PDE which is then used by the core assembly functions.
156  *
157  * The following functions are available for the solution objects \c c and \c u,
158  * where the argument is the field index, i.
159  *
160  * \c c.val(i) - Value of scalar field i, scalar \n
161  * \c c.grad(i) - Gradient of scalar field i, 1st order tensor \n
162  * \c c.hess(i) - Hessian of scalar field i, 2nd order tensor \n
163  * \c c.laplacian(i) - Laplacian of scalar field i, scalar \n
164  * \c c.valP(i) - Value of scalar field i at previous time step, scalar \n
165  * \c c.gradP(i) - Gradient of scalar field i at previous time step, 1st order tensor \n
166  * \c c.hessP(i) - Hessian of scalar field i at previous time step, 2nd order tensor \n
167  * \c c.laplacianP(i) - Laplacian of scalar field i at previous time step, scalar
168  *
169  * \c u.val(i) - Value of vector field i, 1st order tensor \n
170  * \c u.grad(i) - Gradient of vector field i, 2nd order tensor \n
171  * \c u.hess(i) - Hessian of vector field i, 3rd order tensor \n
172  * \c u.valP(i) - Value of vector field i at previous time step, 1st order tensor \n
173  * \c u.gradP(i) - Gradient of vector field i at previous time step, 2nd order tensor \n
174  * \c u.hessP(i) - Hessian of vector field i at previous time step, 3rd order tensor
175  *
176  * Similar functions are available for the test functions. Also, the following tensor operations are useful:
177  *
178  * Tensor operations: \n
179  * \c operator+ - tensor addition \n
180  * \c operator- - tensor subraction \n
181  * \c operator* - single contraction between tensors or scalar multiplication \n
182  * \c double_contract - double contraction of two 2nd order tensors, or
183  * a 4th order tensor and a 2nd order tensor. \n
184  * \c trans( ) - transpose 2nd order tensor \n
185  * \c trace( ) - trace of 2nd order tensor \n
186  * \c det( ) - determinant of 2nd order tensor \n
187  * \c inv( ) - inverse of 2nd order tensor \n
188  *
189  *
190  * The example code here implements the weak form for the Cahn-Hilliard equations, as shown above.
191  *
192  * First, we set the values for necessary parameters, using some predefined material parameters.
193  *
194  * \skip dt
195  * \until tau
196  *
197  * Next, we get the values for the free energy derivatives based on the current quadrature point.
198  *
199  * \skip f_c1c1,
200  * \until F_c2c2(c.val(0),c.val(1));
201  *
202  * Now, we compute the residual in a manner very similar to the analytical form:
203  *
204  * \f{eqnarray*}{
205  * 0 &=& \int_\Omega \left(w_1\frac{c_1 - c_{1_{prev}}}{\mathrm{d}t} +
206  * M\left(\nabla w_1\cdot(f_{,c_1c_1}\nabla c_1) + f_{,c_1c_2}\nabla c_2) + \kappa_1\nabla^2 w_1\nabla^2 c_1\right)\right) dV\\
207  * &\phantom{=}& - \int_{\partial\Omega} \left(w_1j_n +
208  * M\kappa_1\left(\nabla^2c_1(\nabla w_1\cdot\boldsymbol{n}) + \nabla^2w_1(\nabla c_1\cdot\boldsymbol{n})\right)
209  * - \tau(\nabla w_1\cdot\boldsymbol{n})(\nabla c_1\cdot\boldsymbol{n})\right) dS\\
210  * 0 &=& \int_\Omega \left(w_2\frac{c_2 - c_{2_{prev}}}{\mathrm{d}t} +
211  * M\left(\nabla w_2\cdot(f_{,c_2c_1}\nabla c_1) + f_{,c_2c_2}\nabla c_2) + \kappa_1\nabla^2 w_2\nabla^2 c_2\right)\right) dV\\
212  * &\phantom{=}& - \int_{\partial\Omega} \left(w_2j_n +
213  * M\kappa_1\left(\nabla^2c_2(\nabla w_2\cdot\boldsymbol{n}) + \nabla^2w_2(\nabla c_2\cdot\boldsymbol{n})\right)
214  * - \tau(\nabla w_2\cdot\boldsymbol{n})(\nabla c_2\cdot\boldsymbol{n})\right) dS
215  * \f}
216  *
217  * \skip r =
218  * \until //end
219  *
220  * Finally, we include a file that instatiates the template functions \c defineParameters and \c residual. This bit of code
221  * will generally be the same for any problem (unless you decide to use a different automatic differentation library);
222  * the user does not need to modify it.
223  *
224  * \line userFunctionsInstantiation
225  *
226  * The complete implementation can be found at <a href="https://github.com/mechanoChem/mechanoChemIGA/blob/master/initBounValProbs/CahnHilliard_twoSpecies/2D/userFunctions.cc">Github</a>.
227  *
228  * Parameters file: Interface for level 2 users
229  * ==============================
230  *
231  * Now let's look at the parameters file, \c parameters.prm. The advantages of the parameters file are that
232  * these values can be changed without recompiling the code and it can provide a clean interface to the code.
233  * \dontinclude CahnHilliard_twoSpecies/2D/parameters.prm
234  *
235  * The parameters defined in the parameters file overwrite any previous values defined in the \c defineParameters function.
236  * Anything following the pound sign (#) is a comment. A parameter is defined using the syntax:
237  *
238  * \c set \c parameterName \c = \c parameterValue
239  *
240  * There is a set list of variables that can be read from the parameters file. Anything else will be added to
241  * the \c matParam structure with a double number type. Tensor objects can follow the format: 1 x 1 or [1,1] or (1,1),
242  * where the number of components must equal the spatial dimension of the problem.
243  *
244  * In this example file, we begin by specifying the spatial dimension, the geometry dimensions, and the mesh size:
245  *
246  * \skip dim
247  * \until set N
248  *
249  * Next, we define some parameters that are specific to this problem,
250  * so they become elements of \c matParam (see the \c residual and \defineParameters functions above).
251  *
252  * \skip mobility
253  * \until kappa
254  *
255  * We then define time stepping, restart information, output frequency, and spline parameters.
256  *
257  * \skip Time stepping
258  * \until globalContinuity
259  *
260  * Note that we don't need to include all (or even any) of these parameters in this file. We defined default values previously.
261  *
262  * The complete parameters file can be found at <a href="https://github.com/mechanoChem/mechanoChemIGA/blob/master/initBounValProbs/CahnHilliard_twoSpecies/2D/parameters.prm">Github</a>.
