2  * @page example5 Example 5 : Configurational forces
3  * \dontinclude configurationalForces/bending2D/userFunctions.cc
4  *
5  * \htmlonly <style>div.image img[src="configForces2.png"]{width:15cm;}</style> \endhtmlonly
6  * @image html configForces2.png
7  *
8  * In this code, we model the evolving configuration of a material as it undergoes bending.
9  * We solve for two displacement fields. The configurational displacement field represents
10  * the displacement due to evolving material configuration, and is modeled using
11  * a nonconvex strain energy density function.
12  *
13  * \f{eqnarray*}{
14  * \boldsymbol{\Theta} &=& \half(\boldsymbol{\chi}^T\boldsymbol{\chi} - \mathbbm{1})\\
15  * \eta_1 &=& \Theta_{11} + \Theta_{22},\,
16  * \eta_2 = \Theta_{11} - \Theta_{22},\,
17  * \eta_6 = \Theta_{12}\\
18  * \psi^\mathrm{M} &=& \frac{d}{s^2}\left(\eta_1^2 + \eta_6^2\right)
19  * -\frac{2d}{s^2}\eta_2^2 + \frac{d}{s^4}\eta_2^4
20  * + \frac{l^2d}{s^2}|\nabla^0 \eta_2|^2\\
21  * \boldsymbol{D} &=& \frac{\partial \psi^\mathrm{M}}{\partial \boldsymbol{\chi}},\,
22  * \boldsymbol{B} = \frac{\partial \psi^\mathrm{M}}{\partial \nabla^0\boldsymbol{\chi}}\\
23  * \f}
24  *
25  * The standard displacement is governed by an anisotropic St. Venant-Kirchhoff model, where
26  * the anisotropy is dependant on the configurational strain.
27  *
28  * \f{eqnarray*}{
29  * \boldsymbol{E} &=& \frac{1}{2}(\boldsymbol{F}^T\boldsymbol{F} - \mathbbm{1})\\
30  * \alpha_I(\boldsymbol{\chi}) &=& \alpha\Lambda_I(\boldsymbol{\chi}), \, \Lambda_I^2 = \sum_{i=1}^3\chi_{iI}^2\\
31  * \mathbb{C}(\boldsymbol{\chi}) &=& \beta(\mathbbm{1}\otimes\mathbbm{1}) + 2\mu\mathbb{I} +
32  * \sum_{I=1}^3 \left(\alpha_I(\boldsymbol{\chi}) -\beta - 2\mu \right)
33  * \boldsymbol{e}_I\otimes \boldsymbol{e}_I\otimes\boldsymbol{e}_I\otimes \boldsymbol{e}_I\\
34  * \psi^\mathrm{S} &=& \frac{1}{2} \boldsymbol{E}:\mathbb{C}(\boldsymbol{\chi}):\boldsymbol{E} \\
35  * &=& \frac{1}{2}\left[\beta\mathrm{tr}(\boldsymbol{E})^2 + 2\mu(\boldsymbol{E}:\boldsymbol{E}) +
36  * \sum_{I=1}^3 E_{II}^2\left(\alpha_I(\boldsymbol{\chi}) -\beta - 2\mu \right) \right]\\
37  * \frac{\partial \psi^\mathrm{S}}{\partial \boldsymbol{\chi}} &=&
38  * \frac{1}{2} \sum_{I=1}^3 \frac{\alpha}{\Lambda_I}\boldsymbol{\chi}(\boldsymbol{e}_I\otimes \boldsymbol{e}_I) E_{II}^2
39  * \f}
40  *
41  * The total displacement is the sum of the configurational and standard displacements, and
42  * is used to define boundary conditions. We solve the following weak form:
43  *
44  * \f{eqnarray*}{
45  * 0 &=& \int \limits_{{\Omega}_0} \nabla^0\nabla^0\boldsymbol{W}\,\vdots\,\boldsymbol{B} \,\mathrm{d}V_0+\\
46  * &\phantom{=}& +\int \limits_{{\Omega}_0} \nabla^0\boldsymbol{W}:\left[\boldsymbol{D}+J_\chi\left(\mathcal{E} \boldsymbol{\chi}^{-T}
47  * + \frac{\partial \psi^\mathrm{S}}{\partial \boldsymbol{\chi}} \right) \right] \, \mathrm{d}V_0\\
48  * &\phantom{=}& + \int \limits_{{\Omega}_0} \nabla^0\bar{\boldsybmol{w}}:\left(J_\chi\boldsymbol{P} \boldsymbol{\chi}^{-T} \right)
49  * \, \mathrm{d}V_0
50  * \f}
51  *
52  * With the current settings, the following structure is obtained (deformation has been scaled 10x):
53  *
54  * \htmlonly <style>div.image img[src="example5.png"]{width:15cm;}</style> \endhtmlonly
55  * @image html example5.png
56  *
57  * See the paper "A variational treatment of material configurations with application to interface motion and microstructural evolution",
58  * G. Teichert, et al. (Journal of the Mechanics and Physics of Solids, 2017).
59  *
60  * Implementation: Level 1 users
61  * ==============================
62  *
63  * To model evolving material configurations with two displacment fields, we will specify the following through defining user functions: <br>
64  * - Boundary conditions <br>
65  * - Load stepping <br>
66  * - Derived fields for output <br>
67  * - Constitutive model <br>
68  * - Parameter values <br>
69  * - Weak form of the PDE <br>
70  *
71  *
72  * First, we include the header file declaring the required user functions. These functions will be defined in this file.
73  *
74  * \line userFunctions
75  *
76  * Now, we first define any optional user functions. Optional user functions have a default definition that can be
77  * redefined by the user using a function pointer.
78  * This will be done in the \c defineParameters function. The available list of optional user functions includes:
79  * \c boundaryConditions, \c scalarInitialConditions, \c vectorInitialConditions, \c loadStep, \c adaptiveTimeStep, and \c projectFields.
80  * In this example, we redefine the \c boundaryConditions and \c projectFields functions, while using the default functions for the others.
81  *
82  * <b> The \c boundaryConditions function </b>
83  *
84  * This function defines Dirichlet boundary conditions using PetIGA's \c IGASetBoundaryValue function.
85  * The arguments to this function are as follows: the iga object (user.iga),
86  * the "axis" (0, 1, or 2, corresponding to the x, y, or z-axis),
87  * the "side" (0 or 1), the "dof", and the "value" that is to be imposed.
88  * Note that this can only set a uniform value for a degree-of-freedom on any side.
89  * Here, we fix all displacements for both configurational and total displacement fields at x=0 (axis=0,side=0).
90  * We also define vertical configurational and total displacements at x=10 (axis=0,side=1).
91  * Note that the magnitude of the displacement is dependent on the \c scale parameter, which is defined in the next function, \c loadStep.
92  *
93  * \skip template
94  * \until //end
95  *
96  * <b> The \c loadStep function </b>
97  *
98  * This function allows us to update the \c scale parameter and call the \c boundaryConditions function with the updated value
99  * at every time step. This is useful when the Dirichlet boundary condition is too large to allow convergence when applied all at once.
100  * Here, the value for /scale is taken from the current "time".
101  *
102  * \skip template
103  * \until //end
104  *
105  * <b> The \c projectFields function </b>
106  *
107  * If there are field values derived from the solution fields that are of interest, we can compute these
108  * values at each quadrature point and project the value to the nodes. Here, we compute the value \c eta2,
109  * which is a function of the configurational displacement that reflects the local rectangular variant of the crystal structure.
110  *
111  * \skip template
112  * \until //end
113  *
114  * <b> The \c constitutiveModel function </b>
115  *
116  * This function defines the standard and configurational stresses, strain energy densities, and kinematic values that appear in the resiudal.
117  * Note that it is used only in this file (by the \c residual functions),
118  * so it is not a class member function nor does it have an associated function pointer.
119  *
120  * \skip template
121  * \until solutionVectors
122  *
123  * The first part of this function computes variables associated with the standard strain energy density function \f$\psi^\mathrm{S}\f$:
124  *
125  * \f{eqnarray*}{
126  * \boldsymbol{E} &=& \frac{1}{2}(\boldsymbol{F}^T\boldsymbol{F} - \mathbbm{1})\\
127  * \alpha_I(\boldsymbol{\chi}) &=& \alpha\Lambda_I(\boldsymbol{\chi}), \, \Lambda_I^2 = \sum_{i=1}^3\chi_{iI}^2\\
128  * \mathbb{C}(\boldsymbol{\chi}) &=& \beta(\mathbbm{1}\otimes\mathbbm{1}) + 2\mu\mathbb{I} +
129  * \sum_{I=1}^3 \left(\alpha_I(\boldsymbol{\chi}) -\beta - 2\mu \right)
130  * \boldsymbol{e}_I\otimes \boldsymbol{e}_I\otimes\boldsymbol{e}_I\otimes \boldsymbol{e}_I\\
131  * \psi^\mathrm{S} &=& \frac{1}{2} \boldsymbol{E}:\mathbb{C}(\boldsymbol{\chi}):\boldsymbol{E} \\
132  * &=& \frac{1}{2}\left[\beta\mathrm{tr}(\boldsymbol{E})^2 + 2\mu(\boldsymbol{E}:\boldsymbol{E}) +
133  * \sum_{I=1}^3 E_{II}^2\left(\alpha_I(\boldsymbol{\chi}) -\beta - 2\mu \right) \right]\\
134  * \frac{\partial \psi^\mathrm{S}}{\partial \boldsymbol{\chi}} &=&
135  * \frac{1}{2} \sum_{I=1}^3 \frac{\alpha}{\Lambda_I}\boldsymbol{\chi}(\boldsymbol{e}_I\otimes \boldsymbol{e}_I) E_{II}^2
136  * \f}
137  *
138  * \skip mu
139  * \until trans(F)*P
140  *
141  * The second part computes variables associated with the material strain energy density function \f$\psi^\mathrm{M}\f$:
142  *
143  * \f{eqnarray*}{
144  * \boldsymbol{\Theta} &=& \half(\boldsymbol{\chi}^T\boldsymbol{\chi} - \mathbbm{1})\\
145  * \eta_1 &=& \Theta_{11} + \Theta_{22},\,
146  * \eta_2 = \Theta_{11} - \Theta_{22},\,
147  * \eta_6 = \Theta_{12}\\
148  * \psi^\mathrm{M} &=& \frac{d}{s^2}\left(\eta_1^2 + \eta_6^2\right)
149  * -\frac{2d}{s^2}\eta_2^2 + \frac{d}{s^4}\eta_2^4
150  * + \frac{l^2d}{s^2}|\nabla^0 \eta_2|^2\\
151  * \boldsymbol{D} &=& \frac{\partial \psi^\mathrm{M}}{\partial \boldsymbol{\chi}},\,
152  * \boldsymbol{B} = \frac{\partial \psi^\mathrm{M}}{\partial \nabla^0\boldsymbol{\chi}}\\
153  * \f}
154  *
155  * \skip Es
156  * \until //end
157  *
158  * <b> The \c defineParameters function </b>
159  *
160  * The user is required to define the \c defineParameters and \c residual functions. The \c defineParameters defines variables
161  * and functions in the \c AppCtx object. The \c AppCtx object is defined
162  * in the appCtx.h file. This function is used to define any values in \c user that will be needed in the problem.
163  * It is also used to set any function pointers for user functions that we have redefined.
164  *
165  * \skip template
166  * \until void
167  *
168  * We define the mesh, domain, and total applied displacement.
169  *
170  * \skip user.N[0]
171  * \until user.uDirichlet
172  *
173  * We specify the number of vector and scalar solution and projection fields by adding the name of each field to
174  * their respective vector. Here, we have two vector solution fields (the configurational displacement and total displacement) and one scalar projection field
175  * (eta2). We do not use any scalar solution or vector projection fields in this example.
176  *
177  * \skip Displacement
178  * \until "eta2"
179  *
180  * We can specify the polynomial order of the basis splines, as well as the global continuity.
181  * Note that the global continuity must be less than the polynomial order.
182  * Here, we use a quadratic basis function with C-1 continuity because of the gradient elasticity terms.
183  *
184  * \skip polyOrder
185  * \until globalContinuity
186  *
187  * We specify the value \c dtVal, which is a load step in this problem, as well as the \c totalTime, or total loading.
188  * We can also specify a location (iteration number and time) to restart.
189  *
190  * \skip dtVal
191  * \until RESTART_TIME
192  *
193  * Finally, we redirect the desired user function pointers to the \c boundaryConditions, \c projectFields, and \c loadStep functions that we
194  * defined above. This completes the \c defineParameters function.
195  *
196  * \skip boundaryConditions
197  * \until //end
198  *
199  * <b> The \c residual function </b>
200  *
201  * The residual function defines the residual that is to be driven to zero.
202  * This is the central function of the code.
203  * It is set up to follow the analytical weak form of the PDE.
204  * It has a number of arguments that give problem information at the current quadrature point
205  * (see Example 1 and the documentation for the arguments' classes for further information).
206  *
207  * The example code here implements the following weak form (neglecting body force and traction boundary conditions):
208  *
209  * \f{eqnarray*}{
210  * 0 &=& \int \limits_{{\Omega}_0} \nabla^0\nabla^0\boldsymbol{W}\,\vdots\,\boldsymbol{B} \,\mathrm{d}V_0+\\
211  * &\phantom{=}& +\int \limits_{{\Omega}_0} \nabla^0\boldsymbol{W}:\left[\boldsymbol{D}+J_\chi\left(\mathcal{E} \boldsymbol{\chi}^{-T}
212  * + \frac{\partial \psi^\mathrm{S}}{\partial \boldsymbol{\chi}} \right) \right] \, \mathrm{d}V_0\\
213  * &\phantom{=}& + \int \limits_{{\Omega}_0} \nabla^0\bar{\boldsybmol{w}}:\left(J_\chi\boldsymbol{P} \boldsymbol{\chi}^{-T} \right)
214  * \, \mathrm{d}V_0
215  * \f}
216  *
217  * After calling the \c constitutiveModel function to compute the necessary values,
218  * we compute the residual in a manner very similar to the analytical weak form:
219  *
220  * \skip template
221  * \until //end
222  *
223  * Finally, we include a file that instatiates the template functions \c defineParameters and \c residual. This bit of code
224  * will generally be the same for any problem (unless you decide to use a different automatic differentation library),
225  * the user does not need to modify it.
226  *
227  * \line "userFunctionsInstantiation.h"
228  *
229  * The complete implementation can be found at <a href="https://github.com/mechanoChem/mechanoChemIGA/blob/master/initBounValProbs/configurationalForces/bending2D/userFunctions.cc">Github</a>.
230  *
231  * Parameters file: Interface for level 2 users
232  * ==============================
233  *
234  * Now let's look at the parameters file, \c parameters.prm. The advantages of the parameters file are that
235  * these values can be changed without recompiling the code and it can provide a clean interface to the code.
236  * \dontinclude configurationalForces/bending2D/parameters.prm
237  *
238  * The parameters defined in the parameters file overwrite any previous values defined in the \c defineParameters function.
239  * Anything following the pound sign (#) is a comment. A parameter is defined using the syntax:
240  *
241  * \c set \c parameterName \c = \c parameterValue
242  *
243  * There is a set list of variables that can be read from the parameters file. Anything else will be added to
244  * the \c matParam structure with a double number type. Tensor objects can follow the format: 1 x 1 or [1,1] or (1,1),
245  * where the number of components must equal the spatial dimension of the problem.
246  *
247  * In this example file, we begin by specifying the spatial dimension, the geometry dimensions, and the mesh size:
248  *
249  * \skip dim
250  * \until set N
251  *
252  * Next, we define some parameters related to loading:
253  *
254  * \skip Load
255  * \until dtVal
256  *
257  * We then define restart information, output frequency, and spline parameters.
258  *
259  * \skip Restart
260  * \until globalContinuity
261  *
262  * Note that we don't need to include all (or even any) of these parameters in this file. We defined default values previously.
263  *
264  * The complete parameters file can be found at <a href="https://github.com/mechanoChem/mechanoChemIGA/blob/master/initBounValProbs/configurationalForces/bending2D/parameters.prm">Github</a>.
