2  * @page example1 Example 1 : Nongradient, finite strain elasticity
3  * \dontinclude nonGradientMechanics/3D/userFunctions.cc
4  *
5  * To model nongradient, finite strain elasticity, we will specify the following through defining user functions: <br>
6  * - Boundary conditions <br>
7  * - Derived fields for output (e.g. eqivalent stress) <br>
8  * - Constitutive model (via the 1st Piola-Kirchhoff stress) <br>
9  * - Parameter values <br>
10  * - Weak form of the PDE <br>
11  *
12  * First, we include the header file declaring the required user functions. These functions will be defined in this file.
13  *
14  * \line userFunctions
15  *
16  * Now, we first define any optional user functions. Optional user functions have a default definition that can be
17  * redefined by the user using a function pointer.
18  * This will be done in the \c defineParameters function. The available list of optional user functions includes:
19  * \c boundaryConditions, \c scalarInitialConditions, \c vectorInitialConditions, \c loadStep, \c adaptiveTimeStep, and \c projectFields.
20  * In this example, we redefine the \c boundaryConditions and \c projectFields functions, while using the default functions for the others.
21  *
22  * <b> The \c boundaryConditions function </b>
23  *
24  * This function defines Dirichlet boundary conditions using PetIGA's \c IGASetBoundaryValue function.
25  * The arguments to this function are as follows: the iga object (user.iga),
26  * the "axis" (0, 1, or 2, corresponding to the x, y, or z-axis),
27  * the "side" (0 or 1), the "dof", and the "value" that is to be imposed.
28  * Note that this can only set a uniform value for a degree-of-freedom on any side.
29  * Here, we fix all three degrees-of-freedom on the surface at z=0.
30  *
31  * \skip template
32  * \until //end
33  *
34  * <b> The \c projectFields function </b>
35  *
36  * If there are field values derived from the solution fields that are of interest, we can compute these
37  * values at each quadrature point and project the value to the nodes. Here, we compute the equivalent stress
38  * using the 1st Piola-Kirchhoff stress. Scalar values are stored in the \c scalarProjections vector and
39  * vector values are stored in the \c vectorProjections vector.
40  *
41  * \skip template
42  * \until //end
43  *
44  * <b> The \c get1stPiolaKirchhoff function </b>
45  *
46  * This function defines the 1st Piola-Kirchhoff stress. However, it is used only in this file
47  * (by the \c projectFields and \c residual functions), so it is not a class member function nor does it have an associated function pointer.
48  * Additional nonmember functions can be defined in this file, if they are only used by other functions in this same file.
49  *
50  * \skip template
51  * \until //end
52  *
53  * <b> The \c defineParameters function </b>
54  *
55  * The user is required to define the \c defineParameters and \c residual functions. The \c defineParameters defines variables
56  * and functions in the \c AppCtx object. The \c AppCtx object is defined
57  * in the appCtx.h file. This function is used to define any values in \c user that will be needed in the problem.
58  * It is also used to set any function pointers for user functions that we have redefined.
59  *
60  * \skip template
61  * \until void
62  *
63  * Here, we define the mesh by setting the number of elements in each direction, e.g. a 10x10x10 element mesh.
64  *
65  * \skip user.N[0]
66  * \until user.N[2]
67  *
68  * We also define the dimensions of the domain, e.g. a unit cube.
69  *
70  * \skip user.L[0]
71  * \until user.L[2]
72  *
73  * We specify the number of vector and scalar solution and projection fields by adding the name of each field to
74  * their respective vector. Here, we have one vector solution field (the displacement) and one scalar projection field
75  * (the von Mises stress). We do not use any scalar solution or vector projection fields in this example.
76  *
77  * \skip "displacement"
78  * \until "vonMises"
79  *
80  * We can specify the polynomial order of the basis splines, as well as the global continuity.
81  * Note that the global continuity must be less than the polynomial order.
82  * Here, we use a linear basis function with C-0 continuity.
83  *
84  * \skip polyOrder
85  * \until globalContinuity
86  *
87  * We now define the 4th order elasticity tensor.
88  *
89  * \skip double E
90  * \until //end
91  *
92  * Finally, we redirect the desired user function pointers to the \c boundaryConditions and \c projectFields functions that we
93  * defined above. This completes the \c defineParameters function.
94  *
95  * \skip boundaryConditions
96  * \until //end
97  *
98  * <b> The \c residual function </b>
99  *
100  * The residual function defines the residual that is to be driven to zero.
101  * This is the central function of the code.
102  * It is set up to follow the analytical weak form of the PDE.
103  * It has a number of arguments that give problem information at the current quadrature point.
104  *
105  * \skip template
106  * \until &r
107  *
108  * \c dV is a boolean, "true" if \c residual is being called for the volume integral and
109  * "false" if \c residual is being called for the surface integral.\n
110  * \c dS is a boolean, "false" if \c residual is being called for the volume integral and
111  * "true" if \c residual is being called for the surface integral.\n
112  * \c x gives the coordinates of the quadrature point.\n
113  * \c normal gives the unit normal for a surface quadrature point.\n
114  * \c c gives the information (values, gradients, etc.) for the scalar solution fields at the current quadrature point
115  * (see documentation for solutionScalars class).\n
116  * \c u gives the information (values, gradients, etc.) for the vector solution fields at the current quadrature point
117  * (see documentation for solutionVectors class).\n
118  * \c w1 gives the information for the scalar test functions.\n
119  * \c w2 gives the information for the vector test functions.\n
120  * \c user is a structure available for parameters related to the initial boundary value problem (e.g. elasticity tensor).\n
121  * \c r stores the scalar value of the residual for the weak form of the PDE which is then used by the core assembly functions.
122  *
123  * The following functions are available for the solution objects \c c and \c u,
124  * where the argument is the field index, i.
125  *
126  * \c c.val(i) - Value of scalar field i, scalar \n
127  * \c c.grad(i) - Gradient of scalar field i, 1st order tensor \n
128  * \c c.hess(i) - Hessian of scalar field i, 2nd order tensor \n
129  * \c c.laplacian(i) - Laplacian of scalar field i, scalar \n
130  * \c c.valP(i) - Value of scalar field i at previous time step, scalar \n
131  * \c c.gradP(i) - Gradient of scalar field i at previous time step, 1st order tensor \n
132  * \c c.hessP(i) - Hessian of scalar field i at previous time step, 2nd order tensor \n
133  * \c c.laplacianP(i) - Laplacian of scalar field i at previous time step, scalar
134  *
135  * \c u.val(i) - Value of vector field i, 1st order tensor \n
136  * \c u.grad(i) - Gradient of vector field i, 2nd order tensor \n
137  * \c u.hess(i) - Hessian of vector field i, 3rd order tensor \n
138  * \c u.valP(i) - Value of vector field i at previous time step, 1st order tensor \n
139  * \c u.gradP(i) - Gradient of vector field i at previous time step, 2nd order tensor \n
140  * \c u.hessP(i) - Hessian of vector field i at previous time step, 3rd order tensor
141  *
142  * Similar functions are available for the test functions. Also, the following tensor operations are useful:
143  *
144  * Tensor operations: \n
145  * \c operator+ - tensor addition \n
146  * \c operator- - tensor subraction \n
147  * \c operator* - single contraction between tensors or scalar multiplication \n
148  * \c double_contract - double contraction of two 2nd order tensors, or
149  * a 4th order tensor and a 2nd order tensor. \n
150  * \c trans( ) - transpose 2nd order tensor \n
151  * \c trace( ) - trace of 2nd order tensor \n
152  * \c det( ) - determinant of 2nd order tensor \n
153  * \c inv( ) - inverse of 2nd order tensor \n
154  *
155  * The example code here implements the weak form for finite strain elasticity,
156  * \f$\int_\Omega (\nabla{\boldsymbol{w}}:\boldsymbol{P}) dV - \int_{\partial\Omega} (\boldsymbol{w}\cdot\boldsymbol{h}) dS = 0\f$
157  * with the Neumann boundary condtion \f$\boldsymbol{h} = \langle 0,0,1.e9x_1\rangle\f$ on \f$x_3=1\f$.
158  * First, we get the values for \f$\boldsymbol{P}\f$ and \f$\boldsymbol{h}\f$, based on the current quadrature point.
159  *
160  * \skip //Elasticity
161  * \until h[2]
162  *
163  * Now, we compute the residual in a manner very similar to the analytical form
164  *
165  * \skip \int_\Omega
166  * \until //end
167  *
168  * Finally, we include a file that instatiates the template functions \c defineParameters and \c residual. This bit of code
169  * will generally be the same for any problem (unless you decide to use a different automatic differentation library),
170  * the user does not need to modify it.
171  *
